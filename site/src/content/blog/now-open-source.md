<p>Union, the modular ZK interoperability layer for message passing and asset transfers, is now fully open source.</p>
<ul><li><p><a href="https://github.com/unionlabs/union">See the Union GitHub repo</a></p></li></ul>
<p>This opens up the Union repo to community contributions, and enables protocols to permissionlessly integrate and build on Union. Visit the <a href="https://github.com/unionlabs/union">Union repository</a> for a quickstart guide.</p>
<h2>Why Union Is Open Sourcing</h2>
<p>Open source software is key to Web3’s ethos. As compared to the closed-source software created by Web2 incumbents, which has led to opaque security assumptions, monopolistic behavior, and abandoned critical infrastructure, open source aligns with Web3’s ideals of being transparent, collaborative, and fair. That’s why Union Labs has taken the important step of open sourcing the code behind the Union network.</p>
<p>Now, community contributors will be able to submit changes directly to Union, speeding up the pace of innovation and opening it up to a variety of developers, with different skill sets and experiences.</p>
<p>Web3 protocols will now also be able to integrate Union interoperability into their blockchains or applications without needing to collaborate with the Union Labs team. This moves Union toward a permissionless future, where any blockchain or application can add secure zk-powered interoperability to their tech stack in a decentralized fashion.</p>
<p><a href="https://x.com/corcoder">Cor Pruijs</a>, CTO of Union Labs, had this to say:</p>
<blockquote><p><em>Open-sourcing Union is about more than just transparency—it’s about fostering a collaborative ecosystem where everyone can leverage our ZK interoperability to permissionlessly and securely connect to any chain. Due to Union's modular architecture, devs can add support for any execution environment and consensus mechanism.</em></p></blockquote>
<p>Among the components being open sourced are:</p>
<ul><li><p>The <a href="https://github.com/unionlabs/union/blob/main/uniond"><u>Union node implementation</u></a>, using <a href="https://union.build/docs/architecture/cometbls/">CometBLS</a></p></li><li><p><a href="https://github.com/unionlabs/union/blob/main/galoisd"><u>Galois</u></a> zero-knowledge prover</p></li><li><p><a href="https://github.com/unionlabs/union/blob/main/voyager"><u>Voyager</u></a> modular cross-ecosystem relayer</p></li><li><p><a href="https://github.com/unionlabs/union/blob/main/hubble"><u>Hubble</u></a> multi-ecosystem chain indexer</p></li><li><p>Union <a href="https://github.com/unionlabs/union/blob/main/light-clients"><u>light clients</u></a> for various ecosystems</p></li></ul>
<p>Click on the links above to inspect each component, or visit the<a href="https://github.com/unionlabs/union"> <u>GitHub repo</u></a> to explore the whole stack. Be sure to join the<a href="https://discord.union.build/"> <u>Union Discord</u></a> as well, and connect with fellow developers building with Union.</p>
